# 项目结构

## 1. 模块结构（冻结）

```
stratum/
├── stratum-gateway
├── stratum-interface
├── stratum-query
├── stratum-application
├── stratum-domain
├── stratum-infrastructure
├── stratum-job
├── stratum-common
└── stratum-starter
```

## 2. 分层职责（MUST）

### 2.1 Interface

- 负责 HTTP 协议适配、参数校验、响应封装、版本路由
- 不得编写业务规则
- 不得直接访问仓储或中间件客户端

### 2.1.1 Gateway

- 负责统一入口路由、限流、负载均衡与追踪透传
- 不承载业务逻辑
- 不直接访问数据库

### 2.1.2 Query

- 负责读模型查询与结果组装
- 必须通过 Query 基架实现统一分页、排序、过滤
- 不允许写操作

### 2.2 Application

- 负责用例编排与事务边界
- 调用 Domain 模型与仓储接口
- 发布领域事件或集成事件

### 2.3 Domain

- 负责核心业务规则
- 定义聚合、实体、值对象、仓储接口
- 不依赖技术框架

### 2.4 Infrastructure

- 实现仓储、缓存、消息、配置、审计
- 不承载业务决策

### 2.5 Starter

- 负责装配与启动
- 只做配置聚合，不写业务代码

### 2.6 Job

- 负责后台任务调度、任务执行和重试
- 必须通过 ShedLock 防止多副本重复执行
- 任务执行结果必须可观测

## 3. 依赖方向（强制）

- stratum-common：不依赖业务模块
- stratum-gateway：依赖 stratum-common
- stratum-domain：依赖 stratum-common
- stratum-application：依赖 stratum-domain 与 stratum-common
- stratum-query：依赖 stratum-application 与 stratum-common
- stratum-infrastructure：依赖 stratum-domain 与 stratum-common
- stratum-interface：依赖 stratum-application 与 stratum-common
- stratum-job：依赖 stratum-application 与 stratum-common
- stratum-starter：聚合 gateway/interface/query/application/infrastructure/job/common

## 4. 包结构规范（按业务能力）

每个业务能力必须采用以下结构：
- interface.controller / interface.request / interface.response
- gateway.route / gateway.filter / gateway.limit
- query.handler / query.spec / query.dto
- application.command / application.query / application.service
- domain.model / domain.repository / domain.service
- infrastructure.persistence / infrastructure.cache / infrastructure.messaging / infrastructure.outbox
- job.scheduler / job.executor / job.retry

## 5. 并发与事务

- 仓储实现必须无共享可变状态
- 事务只能在 Application 层开启
- 并发更新必须使用乐观锁 version 字段

## 6. 可执行验收

- 模块依赖检查通过
- Domain 无框架依赖
- 任一写用例在 Application 层可定位事务注解
- 仓储实现无成员级请求态字段
- Query 模块不存在写仓储调用
- Job 模块在多副本下不发生重复执行

## 7. Common 扩展复用基线（NODE-A05）

### 7.1 目录与命名约定（MUST）

- `stratum-common` 下新增并固定以下包目录：
	- `common.extension`：`*Extensions` 命名，承载字符串、集合等轻量扩展能力。
	- `common.helper`：`*Helper` 命名，承载时间、分页、校验等通用帮助能力。
	- `common.base`：`Base*` 命名，承载跨模块基础请求/结果模型。
	- `common.enums`：枚举名使用业务无关的基础语义命名（如 `SortDirection`）。

### 7.2 复用约束（MUST）

- application/interface/query 模块禁止重复实现下列基础逻辑：
	- 字符串空白处理与文本判断；
	- 集合判空与不可变拷贝；
	- 分页参数归一化与排序方向解析；
	- 时间戳标准化（UTC/ISO-8601）；
	- 通用参数前置校验（非空、非空白、正数）。
- 若出现同类需求，必须优先复用 `stratum-common` 现有能力；缺失时先补齐 common，再由下游模块调用。

### 7.3 抽样复用点（当前状态）

- `stratum-application`：通过 `ApplicationCommonReuseExample` 复用字符串、时间与校验能力。
- `stratum-query`：通过 `QueryCommonReuseExample` 复用分页、排序、集合与校验能力。
- `stratum-interface`：通过 `ApiResponseFactory` 复用字符串扩展与统一错误响应能力。
