# 查询基架设计

## 1. 目标

统一查询模型、分页排序、过滤表达与结果封装，避免查询接口碎片化。

## 2. 查询基架组成（固定）

- QueryGateway：查询入口
- QuerySpec：过滤与排序表达
- QueryHandler：具体查询实现
- QueryResult：统一结果模型

## 3. 查询契约

每个查询必须定义：
- queryCode
- 入参 DTO
- 排序白名单
- 分页策略
- 输出 DTO

## 4. 分页与排序

- page 从 1 开始
- pageSize 默认 20，最大 200
- sort 仅允许白名单字段

## 5. 过滤规范

- 必须支持精确匹配与区间查询
- 模糊查询必须有字段白名单
- 任意动态 SQL 条件必须参数化，禁止拼接注入

## 6. 查询性能

- 列表查询必须限制最大返回条数
- 大表查询必须使用索引字段过滤
- 复杂报表查询必须异步化为后台任务

## 7. 查询安全

- 查询侧必须接入数据权限过滤
- 非授权字段禁止返回
- 敏感字段默认脱敏

## 8. 可观测性

每个查询必须记录：
- queryCode
- durationMs
- resultCount
- traceId

## 9. 可执行验收

- QueryGateway 路由到对应 QueryHandler 成功
- 分页排序规则符合白名单约束
- 查询接口通过 SQL 注入回归测试
- 慢查询可被日志与指标识别
