# 网关与负载均衡设计

## 1. 目标

通过统一网关实现路由聚合、限流、防护与追踪透传，确保入口治理一致。

## 2. 网关选型（固定）

- 网关实现：Spring Boot Gateway
- 负载均衡位置：项目外处理（裸金属部署时使用 Nginx，或由容器化平台负责）
- 健康检查：由上层负载均衡组件负责主动探测 + 被动失败摘除

## 3. 路由与策略

- 所有外部请求必须先进入网关
- 路由规则按域名与路径前缀划分
- 网关必须透传 traceparent 与 X-Request-Id

## 4. 安全控制

- 网关层必须启用基础限流
- 网关层必须屏蔽非法头与超大请求体
- 敏感管理接口必须限制来源网段

## 5. 负载均衡规则

- 负载均衡不在项目内实现
- 裸金属部署场景使用 Nginx upstream 进行节点摘除与回流
- 容器化部署场景使用编排层 Service/Ingress 进行流量分发与健康检查

## 6. 灰度与发布

- 灰度策略固定为按 Header 路由灰度
- 灰度规则必须可回滚
- 发布期间必须保留旧版本至少一个稳定副本

## 7. 可观测性

网关必须输出结构化访问日志，字段至少包括：
- upstream
- status
- latencyMs
- traceId
- clientIp

## 8. 可执行验收

- 请求全部经网关转发成功
- 上游单节点故障可由项目外负载均衡自动切换
- 限流生效并返回标准错误码
- trace 在网关与应用日志中可关联
