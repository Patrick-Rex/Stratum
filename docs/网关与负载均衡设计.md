# 网关与负载均衡设计

## 1. 目标

通过统一网关实现路由聚合、限流、防护、追踪透传和负载均衡，确保入口治理一致。

## 2. 网关选型（固定）

- 网关实现：Nginx Gateway
- 负载均衡策略：Round Robin
- 健康检查：主动探测 + 被动失败摘除

## 3. 路由与策略

- 所有外部请求必须先进入网关
- 路由规则按域名与路径前缀划分
- 网关必须透传 traceparent 与 X-Request-Id

## 4. 安全控制

- 网关层必须启用基础限流
- 网关层必须屏蔽非法头与超大请求体
- 敏感管理接口必须限制来源网段

## 5. 负载均衡规则

- 上游节点故障时自动摘除
- 节点恢复后自动回流
- 长连接与超时参数统一配置

## 6. 灰度与发布

- 灰度策略固定为按 Header 路由灰度
- 灰度规则必须可回滚
- 发布期间必须保留旧版本至少一个稳定副本

## 7. 可观测性

网关必须输出结构化访问日志，字段至少包括：
- upstream
- status
- latencyMs
- traceId
- clientIp

## 8. 可执行验收

- 请求全部经网关转发成功
- 上游单节点故障可自动切换
- 限流生效并返回标准错误码
- trace 在网关与应用日志中可关联
