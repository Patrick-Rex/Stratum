# ADR-002：选择 Easy-Query 作为 ORM

## 状态
已接受

## 背景
Stratum 需要一个与 JDK 25 + Spring Boot 4.x 兼容的 ORM 框架，同时满足 Domain 层无框架依赖的约束（仓储接口在 Domain 声明，实现在 Infrastructure 中）。

## 决策
使用 Easy-Query（easy-query）作为唯一 ORM，配合 Flyway 做迁移管理。

## 替代方案（已考虑）
- **JPA/Hibernate**：成熟，但注解侵入 Domain 实体，违反 F-001。
- **MyBatis-Plus**：需要 Mapper 接口，与 DDD 仓储模型契合度低。
- **JOOQ**：类型安全，但代码生成复杂，学习成本高。
- **Spring Data JDBC**：轻量，但聚合支持不充分。

## 权衡
Easy-Query 支持将 Infrastructure 实现与 Domain 聚合完全解耦，且对 Easy-Query 生成器限于 Infrastructure 层，Domain 实体不需要框架注解，满足 F-001、F-002。

## 影响
- Easy-Query 仅在 `stratum-infrastructure` 中依赖，Domain 实体保持 POJO 形式。
- 查询构造统一在 QueryHandler 或 RepositoryImpl 中使用 Easy-Query API。
