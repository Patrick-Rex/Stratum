# 后台任务设计

## 1. 目标

后台任务能力用于承载定时任务、延迟任务与批处理任务，要求在多副本部署下保持正确性与可追踪性。

## 2. 技术选型（固定）

- 调度框架：Quartz
- 分布式并发锁：ShedLock
- 执行模型：调度器触发 + 任务执行器分层

## 3. 任务分类

- Cron 任务：固定周期执行
- 一次性任务：指定时间执行
- 批处理任务：分片执行，支持断点续跑

## 4. 任务模型

任务定义必须包含：
- jobCode
- jobType
- cronType 任务必须配置 cron
- onceType 任务必须配置 triggerAt
- timeoutSec
- retryPolicy
- idempotencyKeyRule

## 5. 并发与幂等（强制）

- 同一 jobCode 在同一时间窗只允许一个执行实例
- 任务处理必须具备幂等保障
- 任务超时必须可中断并记录失败原因

## 6. 失败与重试

- 重试采用指数退避
- 超过最大重试次数进入 dead_job 表
- dead_job 必须支持人工重放

## 7. 可观测性

每次执行必须记录：
- jobCode
- scheduleTime
- startTime
- endTime
- durationMs
- result
- traceId

指标必须包含：
- 任务成功率
- 任务平均耗时
- 重试次数
- dead_job 数量

## 8. 安全与治理

- 任务管理接口必须受 RBAC 控制
- 高风险任务必须支持暂停与熔断开关

## 9. 可执行验收

- 多副本部署下同一任务无并发重复执行
- 失败任务可进入 dead_job 并可重放
- 任务日志与 traceId 可完整关联
- 关键任务指标可在 Grafana 展示
