# 消息总线设计

## 1. 目标

构建稳定的异步通信能力，支持领域事件发布、系统解耦与最终一致性。

## 2. 总线选型（固定）

- Broker：RabbitMQ
- 交换机类型：Topic
- 投递语义：至少一次

## 3. 事件模型

消息体必须包含：
- eventId
- eventType
- occurredAt
- aggregateId
- payload
- traceId

## 4. 可靠投递（强制）

- 事务写模型与事件写入 Outbox 必须同事务提交
- Outbox Relay 负责异步投递至 RabbitMQ
- 发送确认失败必须重试并记录

## 5. 消费治理

- 消费端必须手动 ack
- 消费逻辑必须幂等
- 失败消息进入 retry 队列，超限进入 DLQ

## 6. 主题与队列命名

- exchange: stratum.topic
- routing key: 业务域.聚合.事件
- queue: q.业务域.事件
- retry queue: q.业务域.事件.retry
- dlq: q.业务域.事件.dlq

## 7. 顺序与一致性

- 同一 aggregateId 的事件必须按序消费
- 跨聚合流程通过补偿与状态机保证一致性

## 8. 可观测性

指标必须包含：
- 发布成功率
- 消费成功率
- 重试次数
- DLQ 积压

日志必须包含：
- eventId
- messageId
- traceId
- consumerGroup

## 9. 可执行验收

- 数据写入与事件发布满足最终一致性
- 消费端重复投递不会造成副作用
- 重试与 DLQ 流程完整可演练
- trace 可关联发布与消费链路
