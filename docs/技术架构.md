# 技术架构

## 1. 架构目标

Stratum 采用 DDD 单体架构，满足以下目标：
- 单体部署，支持多副本水平扩展
- 业务规则集中在领域层，避免技术侵入
- 安全、可观测、可运维能力默认内建
- 支持后台任务、查询基架、消息总线与工作单元一致性
- 通过网关统一入口并支持负载均衡

## 2. 核心约束（MUST）

- 必须使用 JDK 25 与 Spring Boot 4.0.5+。
- 必须使用 Gradle 9.4.1 构建。
- 必须采用多模块结构并遵循依赖方向。
- Domain 层不得依赖 Spring、ORM、MQ、缓存客户端。
- 应用必须无状态，认证采用 JWT。
- 后台任务必须具备分布式并发保护与幂等。
- 领域事件跨边界投递必须通过 Outbox 保证一致性。
- 查询侧必须通过 Query 基架统一分页、排序、过滤。

## 3. 运行时组件

- Interface：Controller、参数校验、版本路由、异常映射
- Application：用例编排、事务边界、事件编排
- Domain：聚合、实体、值对象、仓储接口、领域服务
- Infrastructure：仓储实现、缓存、MQ、配置中心、审计
- Job Runtime：Quartz 调度器、ShedLock、任务执行器
- Gateway：Nginx 统一入口、限流、路由、灰度与负载均衡

外部组件：
- MariaDB
- Redis（开发单节点，生产 Sentinel）
- RabbitMQ
- Quartz JobStore
- Nacos
- Jaeger + Prometheus + Loki + Grafana

## 4. 部署架构

生产拓扑：
- Nginx
- Stratum App 多副本
- MariaDB
- Redis Sentinel
- RabbitMQ
- Nacos
- 可观测性组件
- 网关高可用组件（Nginx 双实例 + 上层托管负载均衡）

关键策略：
- 无状态化
- 幂等与重试
- 配置中心统一配置

## 5. 技术决策（已确定）

| 主题 | 结论 |
|------|------|
| ORM | Easy-Query |
| 迁移工具 | Flyway |
| 鉴权 | Spring Security + JWT + RBAC |
| API 文档 | OpenAPI 3 + Knife4j |
| 版本控制 | X-API-Version |
| 消息总线 | RabbitMQ Topic + Outbox |
| 后台任务 | Quartz + ShedLock |
| 查询基架 | QueryGateway + QuerySpec + QueryResult |
| 网关 | Nginx Gateway + Upstream LB |

## 6. 可执行验收

架构验收通过标准：
- 包依赖符合 Interface -> Application -> Domain，Infrastructure 仅依赖 Domain/Common
- Domain 中不存在框架相关导入
- starter 模块可独立启动
- 关键外部依赖参数可由 Nacos 下发
- 后台任务在多副本部署下不发生重复执行
- 网关具备限流、鉴权透传与故障节点摘除能力
- 出站消息可通过 Outbox 回放并恢复

## 7. 变更规则

架构变更属于 A 类变更，必须：
1. 先更新本文件与 SDD执行总览
2. 通过评审
3. 再实施代码
