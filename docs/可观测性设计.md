# 可观测性设计

## 1. 观测目标

- 请求可追踪
- 异常可定位
- 性能可量化
- 审计可追责

## 2. Trace 规范

- 协议：W3C Trace Context
- 采集：OpenTelemetry Java Agent
- 导出：OTLP -> Jaeger

强制要求：
- 入站必须提取或生成 trace 上下文
- HTTP 出站必须注入 trace 上下文
- MQ 消息头必须携带 trace 上下文

## 3. 日志规范

### 3.1 结构化日志字段（必填）

- timestamp
- level
- service
- env
- traceId
- spanId
- message

### 3.2 日志分类

- 应用链路日志
- 业务操作审计日志
- 数据变更审计日志

### 3.3 脱敏规则

- 禁止输出 Authorization、Cookie、Set-Cookie
- body 脱敏字段：password、token、idCard、phone、email

## 4. 指标规范

- Micrometer 暴露 /actuator/prometheus
- 采集 HTTP、JVM、连接池、MQ、业务指标
- 用例耗时必须有 Timer
- 采集网关指标、后台任务指标、Outbox 投递指标、查询慢日志指标

## 5. 告警规范

最小告警集：
- HTTP 5xx 比例 > 1% 持续 5 分钟
- P95 > 800ms 持续 5 分钟
- 连接池使用率 > 85%
- MQ 堆积持续增长
- Outbox FAILED 数量持续增长
- 后台任务 dead_job 数量超过阈值
- 网关 429 比例超过阈值

## 6. 排障流程（强制）

1. 从告警定位异常时间窗
2. 在 Grafana/Loki 按 traceId 检索日志
3. 在 Jaeger 查看完整调用链
4. 在审计表确认操作者与业务对象

## 7. 可执行验收

- 任意请求可在日志中按 traceId 查到全链路
- MQ 消费日志与上游 trace 可关联
- 核心指标可在 Grafana 面板展示
- 最小告警集可触发并恢复
- 后台任务执行与重试可按 traceId 与 jobCode 检索
- 网关日志与应用日志可通过 traceId 串联
