# 容器化与部署(Docker Compose)

## 1. 目标

通过标准 compose 文件在本地与测试环境快速拉起依赖，保证开发与联调一致。

## 2. Compose 文件划分（固定）

- docker/compose/docker-compose.dev.yml
- docker/compose/docker-compose.observability.yml

## 3. 开发环境拓扑（固定）

dev 文件必须包含：
- mariadb
- redis（单节点）
- rabbitmq
- nacos
- nginx gateway

observability 文件必须包含：
- jaeger
- prometheus
- grafana
- loki
- promtail

## 4. 端口约定

- app: 8080
- gateway: 80
- mariadb: 3306
- redis: 6379
- rabbitmq: 5672
- rabbitmq 管理台: 15672
- nacos: 8848
- jaeger ui: 16686
- grafana: 3000

## 5. Prometheus 抓取规则

- 应用跑宿主机时抓取 host.docker.internal:8080
- 应用容器化时抓取 app:8080
- 网关指标统一通过 nginx-exporter:9113 抓取
- metrics_path 固定 /actuator/prometheus

## 6. Nginx 负载均衡（生产）

必须透传：
- traceparent
- X-Forwarded-For
- X-Forwarded-Proto
- X-Request-Id

## 7. 生产差异（固定）

- Redis 从单节点切换到 Sentinel
- Nacos 切换到生产配置与鉴权
- 观测组件按容量分离部署
- app 至少 2 副本并接入 upstream
- gateway 必须部署 2 实例并接入上层托管负载均衡

## 8. 启停命令

```bash
docker compose -f docker/compose/docker-compose.dev.yml up -d
docker compose -f docker/compose/docker-compose.observability.yml up -d
docker compose -f docker/compose/docker-compose.dev.yml down
docker compose -f docker/compose/docker-compose.observability.yml down
```

## 9. 可执行验收

- 两套 compose 可独立启动与关闭
- 应用可连通 DB/Redis/RabbitMQ/Nacos
- 应用经 gateway 访问成功
- Prometheus 可抓到应用指标
- Jaeger 可看到应用 trace
- Loki 可检索到结构化日志
- 关闭任一 app 副本后网关可自动切换
