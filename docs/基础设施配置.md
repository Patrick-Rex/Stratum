# 基础设施配置

## 1. 配置目标

本规范定义 MariaDB、Redis、RabbitMQ、Nacos 与 Easy-Query 的统一落地方式。
本规范同时定义 Outbox、后台任务存储与查询基架的基础设施要求。

## 2. 数据库（MariaDB + Flyway）

### 2.1 必选配置

- MariaDB 10.11+
- 连接池 HikariCP
- 数据迁移工具 Flyway

### 2.2 Flyway 规范（强制）

- 脚本目录：db/migration
- 命名：VyyyyMMddHHmm__desc.sql
- 一次变更一个脚本
- 发布顺序：先 migrate，再启动应用

### 2.4 工作单元与 Outbox 表

- 必须创建 outbox_event 表
- outbox_event 必须包含 event_id、event_type、payload、status、retry_count、next_retry_at、created_at
- 写业务数据与写 outbox_event 必须同事务提交

### 2.3 仓储并发约束

- 仓储必须无状态
- 不允许缓存可变查询对象
- 连接由连接池统一管理
- 写冲突统一抛出版本冲突错误码

## 3. 缓存（Redis + Caffeine）

### 3.1 拓扑

- 开发环境：Redis 单节点
- 生产环境：Redis Sentinel
- 本地缓存：Caffeine 作为 L1

### 3.2 缓存策略

- 模式：Cache Aside
- 写流程：旁路更新缓存（先写 DB，再按版本更新缓存；更新失败时降级为删缓存并投递补偿任务）
- TTL：基础 TTL + 随机抖动

### 3.3 读取防护策略（强制）

- 缓存穿透：必须启用空值缓存，空值 TTL 固定 60 秒
- 缓存击穿：必须启用 single-flight（同进程请求合并）+ Redis 分布式锁（跨进程互斥）
- 缓存雪崩：必须启用 TTL 抖动 + 热点 key 分级过期
- 缓存等待：等待首个重建请求完成，默认最长等待 5 秒；超时后按降级策略返回

### 3.4 分布式锁规范（Redis）

- 适用范围：任务调度互斥、热点缓存重建互斥
- 加锁方式：SET NX EX + value 唯一 token，释放锁必须校验 token
- 默认参数：lockTTL=30 秒，acquireWaitTimeout=5 秒
- 锁续期：仅对长任务使用，续期周期不得超过 lockTTL 的 1/3
- 禁止项：禁止使用本地 synchronized 代替分布式锁

### 3.5 缓存键与锁键约定

- 业务缓存键：biz:{domain}:{resource}:{id}
- 空值键：null:{domain}:{resource}:{id}
- 互斥锁键：lock:cache-rebuild:{domain}:{resource}:{id}
- 任务互斥锁键：lock:job:{jobCode}
- 所有缓存键必须显式声明 TTL，禁止永久键

### 3.6 认证相关 key

- auth:rt:{jti}
- auth:sess:{sid}
- auth:user:{userId}:ver

### 3.7 可观测与补偿

- 必须暴露缓存命中率、回源次数、single-flight 合并次数、锁竞争失败次数指标
- 缓存更新失败必须记录 traceId 与 key，并进入补偿队列
- 补偿任务必须幂等，重复执行不得产生副作用

## 4. 消息队列（RabbitMQ）

### 4.1 可靠性要求

- 生产端 confirm
- 消费端手动 ack
- 重试队列 + 死信队列

### 4.2 幂等要求

- 每条消息必须含 messageId
- 消费去重必须使用数据库唯一约束，Redis 仅用于短期缓存加速

### 4.3 Outbox Relay

- Relay 扫描 outbox_event 并投递到 RabbitMQ
- 投递成功后状态置为 SENT
- 投递失败按 next_retry_at 重试，超过阈值置为 FAILED

## 5. 配置中心（Nacos）

### 5.1 配置分组

- stratum.yaml
- stratum-security.yaml
- stratum-observability.yaml
- stratum-job.yaml
- stratum-gateway.yaml
- stratum-query.yaml

### 5.2 动态刷新范围

允许动态：限流阈值、采样率、日志开关、慢调用阈值
禁止动态：数据库连接关键参数

## 6. 后台任务存储

- Quartz JobStore 使用数据库持久化
- 任务失败记录写 dead_job 表
- ShedLock 表必须启用唯一键防并发冲突

## 7. 查询基架基础设施

- QuerySpec 到 SQL 的转换必须参数化
- 分页参数必须统一做上限保护
- 排序字段必须走白名单映射

## 8. 审计落地

- 业务操作日志表：audit_operation_log
- 数据变更日志表：audit_data_change
- 两类日志都必须包含 trace_id、operator_id、created_at

## 9. 可执行验收

- Flyway 在启动前成功执行
- Redis 开发环境单节点可连通
- 热点 key 并发场景下仅一个请求回源，其余请求在 5 秒内复用结果或按降级策略返回
- 分布式锁在 30 秒 TTL 与 5 秒等待参数下可稳定运行并可释放
- RabbitMQ 重试与 DLQ 可触发
- Outbox Relay 可投递并重试
- Nacos 配置可动态刷新指定开关
- Quartz 与 ShedLock 在双实例下无重复执行
- Query 参数化生效且注入测试通过
- 审计表有完整索引与基础字段
