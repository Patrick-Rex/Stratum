# 基础设施配置

## 1. 配置目标

本规范定义 MariaDB、Redis、RabbitMQ、Nacos 与 Easy-Query 的统一落地方式。
本规范同时定义 Outbox、后台任务存储与查询基架的基础设施要求。

## 2. 数据库（MariaDB + Flyway）

### 2.1 必选配置

- MariaDB 10.11+
- 连接池 HikariCP
- 数据迁移工具 Flyway

### 2.2 Flyway 规范（强制）

- 脚本目录：db/migration
- 命名：VyyyyMMddHHmm__desc.sql
- 一次变更一个脚本
- 发布顺序：先 migrate，再启动应用

### 2.4 工作单元与 Outbox 表

- 必须创建 outbox_event 表
- outbox_event 必须包含 event_id、event_type、payload、status、retry_count、next_retry_at、created_at
- 写业务数据与写 outbox_event 必须同事务提交

### 2.3 仓储并发约束

- 仓储必须无状态
- 不允许缓存可变查询对象
- 连接由连接池统一管理
- 写冲突统一抛出版本冲突错误码

## 3. 缓存（Redis + Caffeine）

### 3.1 拓扑

- 开发环境：Redis 单节点
- 生产环境：Redis Sentinel
- 本地缓存：Caffeine 作为 L1

### 3.2 缓存策略

- 模式：Cache Aside
- 写流程：先写 DB，再删缓存
- TTL：基础 TTL + 随机抖动

### 3.3 认证相关 key

- auth:rt:{jti}
- auth:sess:{sid}
- auth:user:{userId}:ver

## 4. 消息队列（RabbitMQ）

### 4.1 可靠性要求

- 生产端 confirm
- 消费端手动 ack
- 重试队列 + 死信队列

### 4.2 幂等要求

- 每条消息必须含 messageId
- 消费去重必须使用数据库唯一约束，Redis 仅用于短期缓存加速

### 4.3 Outbox Relay

- Relay 扫描 outbox_event 并投递到 RabbitMQ
- 投递成功后状态置为 SENT
- 投递失败按 next_retry_at 重试，超过阈值置为 FAILED

## 5. 配置中心（Nacos）

### 5.1 配置分组

- stratum.yaml
- stratum-security.yaml
- stratum-observability.yaml
- stratum-job.yaml
- stratum-gateway.yaml
- stratum-query.yaml

### 5.2 动态刷新范围

允许动态：限流阈值、采样率、日志开关、慢调用阈值
禁止动态：数据库连接关键参数

## 6. 后台任务存储

- Quartz JobStore 使用数据库持久化
- 任务失败记录写 dead_job 表
- ShedLock 表必须启用唯一键防并发冲突

## 7. 查询基架基础设施

- QuerySpec 到 SQL 的转换必须参数化
- 分页参数必须统一做上限保护
- 排序字段必须走白名单映射

## 8. 审计落地

- 业务操作日志表：audit_operation_log
- 数据变更日志表：audit_data_change
- 两类日志都必须包含 trace_id、operator_id、created_at

## 9. 可执行验收

- Flyway 在启动前成功执行
- Redis 开发环境单节点可连通
- RabbitMQ 重试与 DLQ 可触发
- Outbox Relay 可投递并重试
- Nacos 配置可动态刷新指定开关
- Quartz 与 ShedLock 在双实例下无重复执行
- Query 参数化生效且注入测试通过
- 审计表有完整索引与基础字段
