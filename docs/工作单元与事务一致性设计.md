# 工作单元与事务一致性设计

## 1. 目标

统一写事务、仓储提交、事件落盘与提交时机，保证单请求内一致性与跨边界最终一致性。

## 2. 工作单元模型（UoW）

工作单元由 Application 层驱动，必须包含：
- 事务生命周期管理
- 聚合变更跟踪
- 统一提交与回滚
- Outbox 写入

## 3. 事务边界（强制）

- 每个写用例必须在单一工作单元内完成
- Domain 层禁止直接开启事务
- Infrastructure 仅执行持久化，不决定事务策略

## 4. 提交流程

1. 加载聚合
2. 执行业务规则
3. 持久化聚合
4. 写入 Outbox
5. 提交事务
6. 提交后触发非事务性后置动作

## 5. 回滚策略

- 任一步骤失败必须整体回滚
- 回滚后不得产生可见副作用
- 异常必须记录统一错误码与 traceId

## 6. 并发控制

- 聚合写入必须使用乐观锁
- 版本冲突返回 409xx
- 严禁在 UoW 外直接更新聚合根表

## 7. 查询与事务隔离

- Query 默认不参与写事务
- 长查询必须避免锁表
- 报表查询走只读数据路径

## 8. 可执行验收

- 事务失败时数据库与 Outbox 均无脏数据
- 并发写冲突可稳定复现并返回标准错误
- UoW 提交前后日志可完整追踪
- Outbox Relay 可重放未投递事件
